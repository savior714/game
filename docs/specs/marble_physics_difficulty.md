# Spec: Marble Physics Stability & Time-Based Difficulty

## 1. 개요
플레이 시간이 경과함에 따라 게임의 난이도를 높이고(큰 구슬 출현 빈도 증가), 구슬이 서로 겹치는 현상을 물리적으로 개선하여 게임의 시각적 완성도와 도전 욕구를 향상시킴.

## 2. 물리 개선 (Physics Stability)
- **문제**: 단일 충돌 해결 단계에서 중력의 압력으로 인해 구슬이 서로 파고드는 현상 발생.
- **해결**: `_mbStep` 내의 충돌 처리 루프를 다중 반복(Sub-stepping) 처리.
    - 프레임당 4회의 충돌 해결 반복을 수행하여 안정성 확보.
    - 구슬 간의 이격 거리(Separation)를 명확히 유지.

## 3. 동적 난이도 (Dynamic Difficulty)
- **메커니즘**: 게임 시작 후 경과 시간에 비례하여 '난이도 계수(Difficulty Factor)' 산출.
- **산출 로직**:
    - `_mbStartTime`: 초기화 시각 저장.
    - `difficulty = min(1.0, (ElapsedTime / 180,000ms))`: 3분 후 최대치 도달.
- **출현 확률 변화**:
    - **초기 (Diff 0.0)**: Lv.1(45%), Lv.2(35%), Lv.3(20%)
    - **최종 (Diff 1.0)**: Lv.1(25%), Lv.2(35%), Lv.3(40%)
    - 큰 구슬(Lv.3)의 비율을 점진적으로 높여 필드 공간 장악 속도를 가속화함.

## 4. 제약 사항
- 파일 크기 500라인 준수.
- 성능: 저사양 기기에서도 4회 반복 연산이 병목이 되지 않도록 최적화된 루프 구조 유지.

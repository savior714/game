# 수학 연산 패턴 분석 기술 명세 (Pattern Analysis Spec)

## 1. 개요
학습자가 특정 숫자 조합(예: 8+8, 7+6 등)에서 반복적으로 오답을 내는 경우, 이를 하나의 '약점 패턴'으로 정의하고 시스템이 이를 학습하여 집중 출제할 수 있도록 지원합니다.

## 2. 패턴 태크 추출 규칙 (extractPatternTag)

### 2.1 덧셈 (Addition)
- **규칙**: 두 피연산자의 일의 자리 숫자를 추출하여 오름차순으로 정렬 후 조합합니다.
- **태그 형식**: `add_unit_{minDigit}_{maxDigit}`
- **예시**:
    - `68 + 18` -> 일의 자리 `8, 8` -> `add_unit_8_8`
    - `47 + 26` -> 일의 자리 `7, 6` -> 정렬 `6, 7` -> `add_unit_6_7`
    - `125 + 9` -> 일의 자리 `5, 9` -> 정렬 `5, 9` -> `add_unit_5_9`

### 2.2 뺄셈 (Subtraction)
- **규칙**: (피감수 일의 자리)와 (감수 일의 자리)를 순서대로 조합합니다. (교환법칙 성립 안 함)
- **태그 형식**: `sub_unit_{aDigit}_{bDigit}`
- **예시**:
    - `43 - 18` -> `sub_unit_3_8` (받아내림 발생 패턴)
    - `85 - 22` -> `sub_unit_5_2`

## 3. 패턴 기반 문제 생성 (generateForTag)

특정 태그가 주어졌을 때, 해당 패턴을 만족하는 피연산자 `a, b`를 생성합니다.

### 3.1 덧셈 (`add_unit_x_y`)
1. 주어진 `level`에 맞는 `maxSum`, `maxA`, `maxB`를 가져옵니다.
2. `a % 10 === x` (또는 `y`)를 만족하는 `a`를 무작위로 생성합니다.
3. `b % 10 === y` (또는 `x`)를 만족하며 `a + b`가 범위를 넘지 않는 `b`를 생성합니다.

### 3.2 뺄셈 (`sub_unit_x_y`)
1. `a % 10 === x`인 `a`를 생성합니다.
2. `b % 10 === y`이며 `b < a`인 `b`를 생성합니다.

## 4. 기존 시스템과의 통합
- `math/engine.js`의 `stats[op].weaknesses` 객체에 위 태그들을 키로 사용하여 `attempts`, `correct` 정보를 누적합니다.
- `_generateCandidate` 함수에서 30% 확률로 발견된 가장 낮은 정답률의 `weakness` 태그를 선택하여 문제를 생성합니다.

# 과학 문제 데이터베이스 설계 스펙 (Science DB Spec) v2

**최종 수정**: 2026-03-29 — 실제 문항·카테고리는 `science/engine.js` 및 `node science/verify_science_engine.js`가 단일 출처에 가깝다.

## 1. 난이도 설계 원칙 (Kindergarten & Low-Grade Focus)

- **Level 0 (킨더/초등 저학년 핵심)**: 5~7세 수준. 직관적인 감각, 울음소리, 색깔, 명확한 반대 개념.
    - 예: 사과는 빨간색, 고양이는 야옹, 얼음은 차갑다.
- **Level 1 (기초)**: 초등 1~2학년 수준. 일상적인 자연 현상 및 동물 특징.
    - 예: 나무의 구조, 날씨(여름/겨울), 나비의 한살이.
- **Level 2~6**: 초등 3학년 이상 교과과정. (상세 생략)

## 2. 콘텐츠 가이드라인
- **단답형 중심**: 복잡한 문장보다는 핵심 단어 위주로 답을 구성.
- **3지 선다형**: 저학년 아이들의 인지 부하를 줄이기 위해 보기를 3개로 압축.
- **일상 언어 사용**: 과학 전문 용어보다는 아이들에게 친숙한 표현 사용 (예: '기체' 대신 '바람'이나 '숨').

## 3. 자율 기반 검증 (Autonomic Verification)
- **Engine Simulator**: 수천 번의 가상 게임 플레이를 통해 난이도 분포와 안정성을 검증함.
- **Pass Criteria**: 
    - 레벨 0~1의 출제 비중이 전체의 60% 이상일 것 (초기 학습 시).
    - 1,000회 시뮬레이션 중 오류/중단 0회.
    - 최근 10문제 내 중복 발생률 5% 미만.

## 4. 확장 문제 세트 수준별 요약

세부 문항 목록은 코드베이스의 `science/engine.js` 데이터와 동기화한다. 스펙만 갱신하고 데이터를 갱신하지 않는 실수를 방지하기 위해, 변경 후 `node science/verify_science_engine.js`를 실행한다.

# 국어 게임 스펙

**소속**: SDD — 어린이 학습 게임 플랫폼
**상태**: ✅ 완성 | **대상**: 초등 저학년
**최종 수정**: 2026-03-26

---

## 1. 개요

초등 저학년의 국어 능력(받침·맞춤법·어휘)을 게임 형태로 훈련하는 애플리케이션.
공통 플랫폼 아키텍처(로켓 스트릭, 적응형 난이도, 강화학습, 중복 방지)를 그대로 상속한다.
- **중복 방지**: 최근 10문제를 기억하여 연속 출제 차단 (`RECENT_LIMIT=10`)

## 2. 문제 유형 및 카테고리

| 카테고리 | 설명 | 예시 |
|----------|------|------|
| `spelling` | 받침 완성 및 맞춤법 | `나무` vs `나뭇`, `돼요` vs `되요` |
| `antonym` | 반대말 선택 | `크다` → `작다`, `가볍다` → `무겁다` |
| `honorific` | 높임말 변환 | `먹어` → `드세요`, `이/가` → `께서` |

### 2.1 빈칸(클로즈) 표기 규칙

- **표기 통일**: 빈칸은 항상 `'( )'`만 사용한다. (다른 기호/중복 금지)
- **핵심 원칙**: **빈칸 뒤에 조사를 “고정 텍스트”로 붙이지 않는다.**
  - ✅ `아버님( ) 오신다.` / 선택지: `께서`, `이`, `가`, `는`
  - ❌ `아버님( )께서 오신다.` (선택지 `께서`를 넣으면 `께서께서` 중복 가능)
- **허용되는 결합 형태 (3가지)**: 빈칸은 의도에 따라 앞/뒤 텍스트에 “붙을 수도, 떨어질 수도” 있다.
  - **조사 자리(앞말에 붙임)**: 앞말과 조사 사이에는 띄어쓰지 않는다.
    - 예: `할아버지( ) 오신다.` → `할아버지께서 오신다.`
  - **단어 한 칸(뒤 조사에 붙임)**: 명사+조사는 붙여 쓰므로, 빈칸 뒤 조사/어미는 붙여 쓴다.
    - 예: `노란 ( )이 피었습니다.` → `노란 꽃이 피었습니다.`
    - 예: `할머니께서 ( )을 하신다.` → `할머니께서 말씀을 하신다.`
  - **단어 내부(양쪽에 붙임)**: 빈칸이 단어의 일부(음절/형태소)라면 양쪽을 붙여 쓴다.
    - 예: `( )늘은 날씨가 참 좋아요.` → `오늘은 …`
    - 예: `김치( )를 끓이다.` → `김치찌개를 …`
- **UI/데이터 책임**: 선택지는 “빈칸에 들어갈 문자열 전체(조사 포함 가능)”를 대표하며, UI는 `qText`를 **그대로 렌더링**한다(형태소 조합 로직 없음).
- **단일 정답 원칙 (Single Answer Principle)**: 모든 문항은 명확한 단 하나의 정답만 가지며, 중의적 해석이 가능한 오답 후보(Distractor)는 원천 배제한다.
- **중복 정답 방지 (Anti-Ambiguity Distractor)**: 조사(께서/께) 문제에서 '도', '만'과 같이 문맥상 정답이 될 수 있는 보조사는 오답으로 사용하지 않는다. (대신 '를', '의', '에' 등 격이 다른 조사를 배치)

## 4. 스트릭 보상 (20연속) — 보상 룰렛

### 4.1 트리거

- **조건**: 연속 정답(`streak`)이 `LAUNCH_STREAK(=20)`에 도달
- **효과(공통)**:
  - 로켓 발사 애니메이션 수행
  - `globalBoost` 증가 (최대 2)
  - 스트릭은 0으로 리셋

### 4.2 보상 룰렛 옵션 (MVP)

- **룰렛은 1회 회전 후 1개 보상을 확정**하며, 확정된 보상만 실행한다.
- 옵션은 아래 3개로 고정한다.

| id | 표시 | 내용 |
|---|---|---|
| `marble` | 🎮 마블 게임 1판 | `marble` 미니게임(iframe 오버레이) 1판 플레이 |
| `youtube` | 📺 유튜브 15분 보기 | 15분 타이머 오버레이(사용자 휴식용) |
| `snack` | 🍪 간식 하나 고르기 | 간식 선택 안내 오버레이(사용자 보상 안내) |

### 4.3 UX 규칙

- **등장 연출**: 20연속 달성 시 로켓이 **화면 중앙으로 날아가 폭발(펑!)**하는 연출 후, **룰렛이 중앙에 멈춘 상태로 등장**한다.
- **수동 회전**: 룰렛은 자동으로 선택되지 않으며, **사용자가 “룰렛 돌리기”를 눌러야** 보상이 확정된다.
- **재오픈 버튼**: 룰렛을 닫았거나 다시 보고 싶을 때 상단 버튼으로 재오픈 가능.
- **1회 소비**: 보상을 실행(또는 “닫기”)하면 상단 보상 버튼은 다시 숨긴다. (다음 20연속까지 대기)

### 4. 데이터셋 확충 (2026-03-26)

- **대상**: spelling, antonym, honorific 모든 카테고리
- **범위**: 레벨 0(입문) ~ 6(전설) 7단계 체계로 확장
- **안정성 강화**: 중의적 선택지를 제거하고 유일 정답을 보장하도록 문항 정제 (밤하늘, 엄마의 음식 등)
- **목표**: 문항 반복 노출 최소화 및 학습 도메인 확대 (사물, 행동, 기초 문법)
